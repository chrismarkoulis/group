Τα μέλη της ομάδας συναντήθηκαν στις 27 Δεκεμβρίου 2019 για να συζητηθούν:

Οι οντότητες που θα χρησιμοποιηθούν στην εφαρμογή, όπως επίσης και στο ER διάγραμμα και στο DB schema
Οι ρόλοι που θα έχει κάθε μια από τις οντότητες που αφορά κάθε χρήστη
Τεχνολογίες που θα χρησιμοποιηθούν
Ερωτήσεις/Προβλήματα που προέκυψαν

- Στο νέο ER διάγραμμα που αναρτήθηκε διαφαίνεται σε μεγάλο βαθμό η δομή οντοτήτων και δυνατοτήτων αυτών στην εφαρμογή

- Σχετικά με τους ρόλους, αποφασίστηκε

	Ο Guide θα μπορεί να κάνει register με βάση κάποια προσωπικά στοιχεία που καταχωρεί και ένα έγκυρο certificate που θα έχει 
	καταχωρηθεί εκ των προτέρων
	
	Ο Guide θα μπορεί να δημιουργήσει,επεξεργαστεί και διαγράψει μια διαθέσιμη ξενάγηση (Tour), που θα έχει στοιχεία την περιοχή, την ημερομηνία,
	το κόστος και το id του ίδιου.
	
	ο Customer θα μπορεί να κάνει register με βάση κάποια προσωπικά στοιχεία που καταχωρεί, παίρνοντας παράλληλα αυτόματα κάποια credits (μορφή χρημάτων)
	
	Ο Customer θα μπορεί βάση μια διαθέσιμης ξενάγησης να δημιουργήσει/διαγράψει μια ολοκληρωμένη ξενάγηση (Booking)
	Η δημιουργία θα συνεπάγεται αφαίρεση των αντίστοιχων credits του (βάση κόστους του Βooking) και ισόποση προσθήκη στα profits του Guide
	
	O Administration θα μπορεί να επιβλέψει ή και διαγράψει αντικείμενα Guide και Customer και ενδεχομένως θα μπορεί
	να δημιουργήσει και certificates.
	
	
- Σχετικά με τις τεχνολογίες που πρόκειται να χρησιμοποιηθούν, αποφασίστηκε να υπάρχουν μεταξύ άλλων οι:
	
	Hibernate
	Spring MVC
	Spring Security
	JSP
	

- Μερικά από τα ερωτήματα που προέκυψαν είναι:

	-- Τι θα γίνεται στις ολοκληρωμένες ξεναγήσεις (Bookings) σε περίπτωση που διαγράφεται ένας Customer ή και ένας Guide?
	(Θα διαγράφεται όλη η συναλλαγή ακόμα και αν θεωρείται ολοκληρωμένη;)
	(θα διαγράφεται μόνο η τιμή του πεδίου από την οντότητα που διαγράφτηκε; δλδ, να επιτρέπονται null τιμές στην εγγραφή;)
	(θα αντικαθιστάται η τιμή από την οντότητα που διαγράφτηκε με κάποια άλλη;)
	+++
	
	
	-- Πώς μπορεί να υλοποιηθεί και εφαρμοστεί το Spring Security στους ρόλους του Guide, Customer και Αdministration;
	(Στο παράδειγμα που δημιουργήθηκε στην τάξη, δημιουργήθηκε μια οντότητα User, έχοντας στα πεδία της μια λίστα από ρόλους,
	πχ. Role_Admin, Role_User κτλ. Οι οντότητες της εφαρμογής μας είναι τρεις που έχουν - στην παρούσα υλοποίηση - μοναδικό
	ρόλο. Θα χρειαστούν επιπλέον υλοποιήσεις για κάθε οντότητα;)
	
	
	-- Είναι απαραίτητη η χρήση οντότητας Role για να εξυπηρετήσει τις ιδιότητες των Guide, Customer και Administration;
	

Για τα ερωτήματα παραπάνω αποφασίστηκε να γίνει μια έρευνα από κάθε μέλος και κατ' επέκταση, αν δεν βρεθεί κάποια ξεκάθαρη
λύση, η ομάδα να απευθυνθεί στους καθηγητές του Bootcamp
